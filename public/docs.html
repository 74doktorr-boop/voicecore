VoiceCore API Reference

Base URL: https://api.voicecore.dev — Version: v2.0.0

Authentication

All API requests require an API key. Include it in one of these ways:

// Header (recommended)
x-api-key: vc_your_api_key

// Bearer token
Authorization: Bearer vc_your_api_key

// Query param (webhooks)
?apiKey=vc_your_api_key

Quick Start

Create your first voice assistant in 30 seconds:

curl -X POST https://api.voicecore.dev/api/assistants \
  -H "x-api-key: vc_your_key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Assistant",
    "voice": "nova",
    "model": "gpt-4o-mini",
    "firstMessage": "Hello! How can I help you?",
    "language": "es"
  }'

Assistants

GET/api/assistantsList all assistants
GET/api/assistants/:idGet assistant details
POST/api/assistantsCreate assistant
PUT/api/assistants/:idUpdate assistant
DELETE/api/assistants/:idDelete assistant

Create Assistant Body

FieldTypeDescription
name requiredstringDisplay name
voicestringVoice ID (default: nova)
modelstringLLM model (default: gpt-4o-mini). Use groq/llama-3.3-70b for ultra-low latency
languagestringISO language code (default: es)
systemPromptstringSystem instructions
firstMessagestringGreeting message spoken on call start
ttsProviderstringopenai | cartesia | elevenlabs | google
toolsarrayFunction tools array (OpenAI format)

Webhooks

VoiceCore sends webhook events for call lifecycle events:

// Webhook payload
{
  "event": "call.ended",
  "timestamp": "2026-05-18T02:00:00Z",
  "data": {
    "callId": "uuid",
    "duration": 180000,
    "turnCount": 12,
    "transcript": [...],
    "cost": { "total": 0.1149 }
  }
}

Events: call.started, call.ended, call.failed, assistant.created
