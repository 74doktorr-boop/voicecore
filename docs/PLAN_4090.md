## DÍA 1 — Primera voz funcional (tú mismo)

**Objetivo: llamada telefónica real con tu voz en euskera.**

**Mañana (09:00–13:00):**
1. Grabar audio de referencia siguiendo `voices/GUION_GRABACION.md`
   - 20 segundos, habitación cerrada, móvil máxima calidad
   - Transcribir exactamente lo que dices → meterlo en `config/voice_profiles.json`

2. Arrancar el servidor TTS:
   ```bash
   python tts_server.py
   # → "Server ready — 0.0.0.0:8000"
   ```

3. Prueba de síntesis manual:
   ```bash
   curl -X POST http://localhost:8000/synthesize \
     -H "Content-Type: application/json" \
     -d '{"text":"Egun on, klinika dentala. Ane naiz, nola lagundu dezaket?","voice_id":"ane"}' \
     --output test.pcm

   # Escuchar (convertir PCM → WAV):
   python -c "
   import wave, struct
   data = open('test.pcm','rb').read()
   with wave.open('test.wav','wb') as w:
       w.setnchannels(1); w.setsampwidth(2); w.setframerate(24000)
       w.writeframes(data)
   "
   ```

4. Si suena bien → arrancar `npm start` en voicecore y probar llamada vía Twilio

**Tarde (15:00–21:00):**
5. Afinar parámetros en `tts_server.py` si hace falta:
   - `F5_NFE_STEP=64` para más calidad (más lento: ~2s extra)
   - `cross_fade_duration=0.05` si hay artefactos entre frases
6. Medir latencia end-to-end con `node src/utils/benchmark.js` (si existe)
7. Configurar ngrok o equivalente para exponer el TTS a Twilio en la red

## DÍA 4–5 — Fine-tuning F5-TTS (OPCIONAL pero recomendado)

**Solo si la calidad zero-shot no convence al 100%.**

```bash
# Instalar herramientas de fine-tuning F5-TTS
pip install accelerate wandb

# Estructura de datos de entrenamiento:
# data/ane/
#   ane_01.wav + ane_01.txt
#   ane_02.wav + ane_02.txt
#   ... (mínimo 20 pares, ideal 100+)

# Lanzar fine-tuning (RTX 4090, ~4h para 20min de audio)
python finetune_f5tts.py \
  --dataset data/ane \
  --output checkpoints/ane_finetuned \
  --epochs 50 \
  --batch_size 4
```

*(El script de fine-tuning lo generamos cuando tengas los audios)*

## Señales de alerta a vigilar

| Síntoma | Causa probable | Solución |
|---------|---------------|----------|
| `RuntimeError: CUDA out of memory` | Otra app usa la GPU | Cerrar apps, reducir `F5_NFE_STEP` |
| Latencia >3s en síntesis | `nfe_step` muy alto | Bajar a 32 |
| Voz suena robótica | `ref_text` no coincide con el audio | Retranscribir exactamente |
| Acento suena castellano | F5-TTS base sin fine-tune | Hacer fine-tuning (Día 4-5) |
| Node.js → circuit breaker abierto | Servidor TTS crasheado | Reiniciar `python tts_server.py` |
